# koden-mu

株式会社光電製作所の簡易型気象レーダー試作機（プロジェクト番号 #1800077）で取得したIQデータを解析・可視化するPythonスクリプト集です。

## スクリプト概要

| スクリプト | 説明 |
|---|---|
| `scripts/process_iq.py` | IQデータ（`.bin`）を読み込み、受信電力マップの描画とNPZファイルへの保存を行う |
| `scripts/plot_doppler_sample.py` | NPZファイルを読み込み、指定したゲートのドップラースペクトルをプロットする |

## 環境構築

[uv](https://docs.astral.sh/uv/getting-started/installation/) を使った環境構築を推奨します。

```bash
uv sync
```


## 使い方

### 1. IQデータの処理（`process_iq.py`）

```bash
uv run scripts/process_iq.py <IQデータファイル.bin>
```

実行すると、入力ファイルと同じ名前で以下の2ファイルがカレントディレクトリに生成されます。

| 出力ファイル | 内容 |
|---|---|
| `<入力ファイル名>.png` | PPI形式の受信電力マップ（解像度 300 dpi） |
| `<入力ファイル名>.npz` | 処理済みデータを格納したNumPy圧縮ファイル |

#### NPZファイルの内容

| キー | shape | 説明 |
|---|---|---|
| `power` | `(10, 2048)` | 受信電力 [dBm]。仰角 × レンジの2次元配列 |
| `x` | `(11, 2049)` | 受信電力プロット用のX座標 [km]。`pcolormesh` の境界グリッドとして +1 大きい |
| `y` | `(11, 2049)` | 受信電力プロット用のY座標 [km]。同上 |
| `doppler` | `(10, 2048, 200)` | ドップラー電力スペクトル [dB]。仰角 × レンジ × 速度ビンの3次元配列 |
| `velocity` | `(200,)` | ドップラー速度軸 [m/s]。最大不曖昧速度の範囲でfftshift済み |

### 2. ドップラースペクトルの表示（`plot_doppler_sample.py`）

```bash
uv run scripts/plot_doppler_sample.py <NPZファイル.npz>
```

デフォルトでは仰角インデックス `0`、レンジインデックス `0` のドップラースペクトルを表示します。
他のゲートを表示するには `plot_doppler_sample.py` 内の以下の行を編集してください。

```python
ax.plot(velocity, doppler[0, 0, :])
#                         ↑  ↑
#                         |  レンジインデックス
#                         仰角インデックス
```

## IQデータファイルについて

### ファイル命名規則

IQDatCapture によって以下の形式でファイルが生成されます。

```
<prefix>_yyyymmddHHMMSS_NNNN.bin
```

| 部分 | 説明 |
|---|---|
| `<prefix>` | ユーザーが指定したファイル名 |
| `yyyymmddHHMMSS` | 保存開始時のPC時刻 |
| `NNNN` | ファイル番号（0始まり、最大ファイル数 − 1 まで連番） |

> **注意**: 1ファイル目（`_0000.bin`）は設定した駆動範囲のデータが揃っていない場合があります。解析には **2ファイル目（`_0001.bin`）以降** を使用してください。

### バイナリフォーマット

IQデータはパルス圧縮後の生IQ信号を対数化・量子化したバイナリファイルです。

**ブロック構造**

1ブロックのサイズは 1040 Byte で、ヘッダー部とデータ部から構成されます。
方位ごとに 8 ブロック（I/Q × 分割番号 0〜3）で 1 スウィープを構成します。

```
1スウィープ (8ブロック)
├── I信号 分割番号0  [header 16B + data 1024B]
├── Q信号 分割番号0  [header 16B + data 1024B]
├── I信号 分割番号1  [header 16B + data 1024B]
├── Q信号 分割番号1  [header 16B + data 1024B]
├── ...
└── Q信号 分割番号3  [header 16B + data 1024B]
```

**ヘッダー部（16 Byte、リトルエンディアン）**

| フィールド | サイズ | 内容 |
|---|---|---|
| パラメータ番号 | 4 Byte | 未使用 |
| IQフラグ | 2 Byte | `0` = I信号、`1` = Q信号 |
| 方位角 | 2 Byte | 値 × 0.25 = 方位角 [deg] |
| 分割番号 | 4 Byte | 0〜3（レンジ方向を512ゲートずつ4分割） |
| スウィープ番号 | 4 Byte | 0〜35999 の連番 |

**データ部（1024 Byte）**

512 サンプル × 2 Byte（符号付き16bit整数、リトルエンディアン）。
パルス圧縮後のIQ信号は対数化（底2）された固定小数点形式で記録されています。

値 `d` からパルス圧縮後の振幅を復元する手順：

```
x = d / 2^9          # 固定小数点への変換（上位6bit: 整数部、下位9bit: 小数部）
X = 2^|x|            # 真数化
振幅 = sign(x) × X   # 符号付与
```

復元後の振幅からアンテナ入力端の電力を求める式：

```
P [dBm] = 20 * log10(sqrt(i^2 + q^2)) - offset
```

**オフセット値（キャリブレーション係数）**

| パルス種別 | レンジ範囲 | offset [dB] |
|---|---|---|
| 短パルス | 1〜213 bin（0〜4.24 km） | 395.8 |
| 長パルス | 214〜2048 bin（4.26〜40.94 km） | — |

## レーダーパラメータ

| パラメータ | 値 |
|---|---|
| 送信周波数 | 9.4 GHz |
| PRF | 2400 Hz |
| レンジゲート数 | 2048 |

| 方位分解能 | 1.0 deg/step |
| コヒーレント積分数 | 200 ヒット |
| 最大不曖昧速度 | ≈ ±38 m/s |
